# New Release — 1.7 ([RU](/ru/changelogs/8305)/[EN](https://wiki.eyeauras.net/en/changelogs/8305))

A lot of work went into making **Behavior Trees** and the entire scripting system more flexible. We can now finally build bots using **only** Macros and Behavior Trees, without needing to rely on Auras.

Combining two complex systems (Auras and BT) made the logic harder to follow, so the new nodes should make things much clearer. And this is not the end of it — more changes are coming soon, especially around preview-related functionality.

---

## Screen Capture — **BREAKING CHANGE**

For many years, all `Capture` triggers (Image/Color/Text/ML) required the **`Target Window`** parameter — EyeAuras worked **only** with windows.

Starting with this version, we are trying a different approach:

- **If `Target Window` is not specified, EA will capture the primary screen.**

### What does this mean in practice?

- Previously, clearing the `Target Window` field would effectively stop those triggers from working. Now it simply switches them to primary screen capture mode. I strongly recommend using [Enabling Conditions](/features/enabling-conditions) to control capture. This is important! **Check your old triggers to make sure there are no "test" or "forgotten" ones — they will start consuming CPU.**
- One less setup step if you are configuring something for personal use: in many games running in *Borderless Windowed* mode, or similar fullscreen-like modes, specifying `Target Window` is no longer required.
- Once the window dependency is removed, working with mouse coordinates becomes simpler — [every coordinate becomes absolute](/features/coordinate-system).
- Anything related to clicking objects on the screen should continue to work as before. If you notice any issues, please report them.

> **IMPORTANT!** Multi-monitor setups are not supported yet. Only the primary screen is captured. Support will be expanded in the future.

##  Behavior Trees & Macros Improvements

- [**Major Macros changes**](https://wiki.eyeauras.net/en/changelogs/8174): macro flexibility has been significantly expanded.
- **New CV-based nodes**: computer vision nodes that act as trigger equivalents. For simple bots, triggers are no longer required!
  - [MLSearch together with MLFindClass](https://wiki.eyeauras.net/en/changelogs/8193) — works with ML models and lets you filter ML search results. Combined with MouseMove improvements, you can build a point-and-click bot in minutes.
  - [PixelSearch](https://wiki.eyeauras.net/en/changelogs/8185) — finds a pixel of the specified color.
  - [ImageSearch](https://wiki.eyeauras.net/en/changelogs/8202) — finds an image.
  - [ColorCheck](https://wiki.eyeauras.net/en/changelogs/8202) — checks whether the color of a pixel/area matches the expected value.
- **New logic nodes**: let you control execution flow.
  - [Interrupter](https://wiki.eyeauras.net/en/changelogs/8227) — conditionally interrupts nodes that are already running.
  - [Timeout](https://wiki.eyeauras.net/en/changelogs/8227) — sets the maximum time a child node may remain in the `Running` state.
  - [CheckKeyState](https://wiki.eyeauras.net/en/changelogs/8227) — checks the current state of the specified key.
  - [IfThenElse](https://wiki.eyeauras.net/en/changelogs/8202) — `if..then..else` as a BT node, useful for users who are just getting familiar with BTs.
- [**Undo/redo in BT**](https://wiki.eyeauras.net/en/changelogs/8297): you can now undo/redo node additions and deletions with `Ctrl+Z` / `Ctrl+Y`.
- [**Hotkey activation**](https://wiki.eyeauras.net/en/changelogs/8227): BTs and Macros can now be started directly with hotkeys.
- [**Login Widget**](https://wiki.eyeauras.net/en/changelogs/8055): a simple widget that can be used in mini-apps for EyeAuras-based authentication.
- [**PopOut View**](https://wiki.eyeauras.net/en/changelogs/8077): a read-only popup window for BT/Macro testing and debugging.
- [**BT variable support**](https://wiki.eyeauras.net/en/changelogs/8185): nodes like `MouseMove` can now use `CvLastFoundRegion` from CV searches.

##  C# Scripting System

- [**Precompilation and script persistence**](https://wiki.eyeauras.net/en/changelogs/7994): scripts are compiled ahead of time, reducing runtime delays. Compiled binaries can be stored to avoid recompilation after restart.
- [**API File Provider**](https://wiki.eyeauras.net/en/changelogs/8097): scripts can inject and load additional files (`.css`, `.js`, `.md`, `.dll`) via `IScriptFileProvider`.
- [**NuGet package support**](https://wiki.eyeauras.net/en/changelogs/8047): packs automatically fetch and package required NuGet dependencies for a better offline and startup experience.
- [**Keybind attributes**](https://wiki.eyeauras.net/en/changelogs/8162): scripts can bind methods to hotkeys via `[Keybind]`.
- [**Dependency Injection improvements**](https://wiki.eyeauras.net/en/changelogs/8162): cleaner script structure thanks to `[Dependency]` or init properties for services such as `SendInput`.
