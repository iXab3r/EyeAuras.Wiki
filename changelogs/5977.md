# New Release!

Here’s a summary of the changes made to the app over the **last 4 months**.

A lot of work went into this release: improved C# scripting (for Triggers/Actions and Overlays), an upgraded **Send Sequence** system (which now fully replaces **Send Input** in terms of functionality), a better configuration subsystem, and much faster, more efficient **Image Effects** that can now downscale images for better performance.

To support the needs of the new and rapidly growing machine learning community, I also developed a separate app to make it easier to create and train Yolo models, which are fully supported by the **ML Search** feature. In my experience, this is much faster than preparing data manually, even though the app is still in an early alpha stage. For more information, see: [Getting Started | EyeAuras Wiki](https://wiki.eyeauras.net/en/CVATAAT/getting-started).

## Transition to a new UI technology

One of the main ongoing initiatives is the move to a new UI technology. This change is intended to solve current responsiveness issues in the software, such as switching between auras or creating new triggers/actions.

Although this is a slow transition and may take another **6–12 months**, the expected result should bring major improvements. You can already see this in the new **Import/Export** and **Bindings** dialogs, which now use this technology. Over the coming months, more parts of the app will be replaced and rewritten one by one (Aura Tree, I’m looking at you).

## What’s next?

**Performance:** The main focus of the next update is performance. I’ve completely redesigned the configuration system; it is currently in testing and should make app startup and overall operation much faster, especially on lower-end PCs. As mentioned above, I’ll also continue rewriting slower parts of the app, such as the Aura Tree, to improve the overall user experience.

**Behavior Trees:** Currently in development and testing, Behavior Trees are the next major feature to look forward to. They fundamentally change how you work with the app. They also use the new UI technology, so they are much more responsive :)

Stay tuned for updates, and thank you for your continued support and for using the software!

## New features and improvements

- **Send Sequence 2.0**: Combines all action types into an improved input sequencing system.
- **New C# Scripting engine**: Now available in Triggers/Actions (alpha stage), expanding capabilities with new and improved [APIs](https://docs.eyeauras.net/)
- **Bindings improvements**: The Bindings editor has been rewritten for better performance and usability. It is much, much, much more responsive and flexible.
- **Improved scripting with Visual Studio/Rider compatibility**: Export and save your code as a C# project for more advanced development workflows.
- **Import/Export dialog redesign**: More responsive, more detailed, and more user-friendly. The Sync dialog is expected to be updated in one of the upcoming releases.

## Performance and usability improvements

- **Network messaging improvements**: Infrastructure changes for better performance and reduced latency.
- **Overlay UI and general UI performance**: Significant changes for better performance and responsiveness.
- **General performance improvements**: Focused on improving overall user experience and software responsiveness.
- **Aura Tree responsiveness**: Interface improvements for smoother interaction with the Aura Tree.

## Other improvements

- **Image scaling in the Recalculate Image effect**: Now supports scaling on both the X and Y axes. You can now also downscale images using effects to improve performance, and the app will still be able to calculate the correct coordinates of the detected image and click it. This works for **ALL** trigger types: **ML, Color, and Image**.
- **New Image Effects**: Added color range replacement and noise removal effects for better image analysis.
- **ML Search improvements**: Significant performance improvements and new features in ML Search. This is especially relevant for scripting, since scripts unlock the full potential of ML.
