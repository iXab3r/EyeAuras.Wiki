## Added a License Agreement

To address the legal side of things, I added a mandatory license agreement. It states that the user is solely responsible for complying with the license agreements of any applications they automate.

As many of you know, games often explicitly state that automating actions in any form is prohibited and may result in a ban. Please keep this in mind.

## Improvements to Window Activation

For EyeAuras to work effectively, the window receiving input usually needs to be active. If it is not, EyeAuras includes several methods to automatically bring it to the foreground.

At first glance, this may seem like a simple one-line API call. However, due to various Windows-specific quirks, it is not that straightforward. As a result, I implemented several mechanisms to improve the chances of success.

In this release, I added more code to improve reliability, especially in edge cases and with applications that resist automated activation.

**In short:** if you are having trouble with window activation—for example, if a window does not become active when you send input—please contact me, as this version includes adjustments in that area.
