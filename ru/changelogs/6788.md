# Права доступа к папкам (ранняя альфа)
Из-за особенностей скриптов, создаваемых в EyeAuras, иногда бывает полезно запретить пользователю менять что-либо в распространяемом вами паке аур. Это может понадобиться, чтобы избежать случайной неправильной настройки или, в некоторых случаях, просто скрыть внутреннюю реализацию вашего механизма.

Именно для этого были добавлены права доступа на уровне папок. Они позволяют любому авторизованному пользователю задать ограничения для конкретной папки, а программа защитит её содержимое от внешнего просмотра и изменений.

Сейчас разрешения состоят из 3 частей:
- **Folder Owner** — пользователь, который изначально назначил это разрешение. Пока защита не снята, у него есть полный контроль над папкой: он может делать с ней что угодно, включая снятие ограничений.
- **Folder Password** — способ временно повысить права текущего пользователя до уровня **Folder Owner**. То есть любой пользователь, знающий этот пароль, может стать **Folder Owner** до тех пор, пока папка снова не будет заблокирована **ИЛИ** пока программа не будет перезапущена.
- **Permission Level** — определяет, что именно могут делать не-владельцы с данными в этой папке: изменять / просматривать / запускать. Подробнее — на странице [Permission model](/permission-model).

Сейчас основной способ задать уровень доступа — через контекстное меню.

# Новая форма входа / профиля пользователя (ранние версии)
Внешний вид формы входа был полностью переработан. Сейчас в ней уже отображаются статус вашей текущей лицензии (free/pro), оставшееся время, а в ближайшем будущем там также появится список отдельно приобретённых модулей / подписок. Подробнее об этом механизме я расскажу позже этим летом. В целом идея в том, чтобы пользователи могли обмениваться скриптами друг с другом.

![](https://i.imgur.com/RZbht3F.png)

И ещё: предупреждение, которое показывается при превышении лимита на количество аур для Free-аккаунтов (а это 10), стало гораздо заметнее. Надеюсь, это сбережёт пользователям немного нервов — раньше, если вы забыли продлить лицензию и запускали программу, было не так просто понять, почему скрипты не работают.

Теперь внизу дерева будет показываться большое предупреждение, а ауры, которые неактивны по техническим причинам, будут зачёркнуты.

![](https://i.imgur.com/Em2GXvR.png)
