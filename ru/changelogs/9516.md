# 1.9.9516 [RU](/ru/changelogs/9516)/[EN](/en/changelogs/9516)
## AI - Agent Guidance
Добавлена страница `Agent Guidance` в AI-настройках.

![Guidance Pack Actions](https://s3.eyeauras.net/media/2026/04/EyeAuras_R4JvFSFixc.png =x600)

Если совсем просто: EyeAuras теперь умеет подготовить для внешних AI coding agents переносимую папку `guidance-pack`, в которой лежат актуальные docs, примеры, SDK-заметки, metadata и MCP status.

Это нужно для сценариев, где вы открываете EyeAuras-проект, экспортированный workspace или C# проект во внешнем агенте вроде `Codex`, `Claude Code`, `Cursor` и т.п. Агенту больше не нужно каждый раз вручную объяснять, что такое EyeAuras и где искать правильный контекст.

Важно: EyeAuras теперь работает именно на уровне папки. Ваш проектный `AGENTS.md` не меняется и не перезаписывается. Внутри самой папки guidance pack есть свой `AGENTS.md`, с которого агент может начать.

Что можно сделать прямо из окна:

* пересобрать guidance pack
* открыть папку с готовыми AI-facing файлами
* сохранить pack в zip-архив
* скопировать весь pack в `generated/ai` через выбранный `.csproj` или `.sln`

Внутри pack лежит `AGENTS.md`, `README.md`, `docs/`, `manifest.json` и, если MCP status был записан, `mcp-status.json`.

![Guidance Pack Folder](https://s3.eyeauras.net/media/2026/04/EyeAuras_pW0O4jRT9q.png =x600)

Папка `generated/ai` управляется EyeAuras и при повторном копировании заменяется целиком.

## AI - MCP helper
В этом же окне появился более удобный `MCP helper`.

MCP - это локальный мостик, через который AI-агент может получить точечный контекст от EyeAuras. Docs остаются основным источником, а MCP стоит включать тогда, когда агенту нужно спросить EyeAuras о текущем состоянии программы, окнах или доступных объектах/символах.

Для обычных вопросов по wiki это можно спокойно игнорировать. Для внешних coding agents и сложных scripting-задач - наоборот, может быть очень полезно.
