# Image / Text / Color Search

Эта группа триггеров основана на захвате и анализе изображения. Секция с выбором окна и настройками захвата общая, а вот настройки анализа уже разные.

### Настройки

- **Target FPS** - указывает насколько часто нужно выполнять захват и обработку изображения. Чаще всего реальный FPS будет ниже, так как на анализ (распознавание текста, поиск изображения) тратится довольно много времени и ресурсов
- **Mouse Cursor** - некоторые способы захвата поддерживают включение/выключение захвата курсора, эта настройка контролирует данное поведение
- **Settings => Max Capture FPS** - данная настройка ограничивает FPS для всех триггеров

### Способы захвата

- **Windows Graphics** - основан на официальном Windows API для захвата, из минусов - на версиях младше **10.0.19041.0** у захватываемого окна появляется желтая рамка. В более современных версиях ОС этой рамки уже нет.
- **Shared Surface** - основан на копировании участка памяти GPU где хранится изображение окна. В целом крайне похож на Windows Graphics, но более низкоуровневый, соответственно, менее надежный
- **Desktop Duplication** - основан на копировании участка памяти GPU где хранится изображение всего рабочего стола. Дальше это изображение урезается до нужного региона.
- **Print Window** - использует старый API, в целом довольно универсальный и производительный метод, но не умеет захватывать некоторые виды окно, к примеру, браузеры
- **Copy Device Context** - один из legacy способов захвата, оставлен исключительно в академических целях, в большинстве ситуаций не нужен
- **Copy From Screen** - самый медленный и самый неэффективный способ захвата, который по факту делает скриншот экрана и дальше работает с полученным изображением

Основным способом захвата считается **Windows Graphics**, у него оптимальное соотношение CPU/RAM, однако в некоторых случаях он может блокироваться играми. В таком случае можно воспользоваться списком ниже для выбора.

###### Отсортированные по производительности

Windows Graphics > Shared Surface > Desktop Duplication > Print Window > Copy Device Context

###### Отсортированные по потреблению памяти (выше место - меньше потребление):

Print Window > Windows Graphics > Shared Surface > Copy Device Context > Desktop Duplication > Copy From Screen
