# Триггер поиска изображения

Триггер Image Search в EyeAuras позволяет указать изображение, которое EyeAuras будет искать в захваченном окне или в выбранной области этого окна. Поддерживается в том числе поиск изображений с прозрачностью, поэтому этот триггер подходит для самых разных сценариев. В EyeAuras уже используются различные оптимизации для ускорения поиска и снижения нагрузки на систему, но для лучшей производительности всё равно рекомендуется максимально сужать область поиска.

## Целевое окно

В этом поле указывается окно, в котором EyeAuras должен искать изображение. Для этого используется [_Window Match Expression_](https://wiki.eyeauras.net/e/ru/window-match-expressions).

## Изображение

Здесь загружается изображение, которое EyeAuras будет искать в целевом окне. Изображение сохраняется внутри триггера и может быть загружено из файла. После загрузки сам файл больше не влияет на работу EyeAuras. Изображения с прозрачностью также поддерживаются: при сравнении EyeAuras игнорирует области, помеченные как прозрачные.

## Порог совпадения

Порог совпадения определяет, насколько найденное содержимое в целевом окне должно быть похоже на выбранное изображение. Если процент сходства достигает или превышает заданный порог, триггер активируется. Если становится ниже — деактивируется.

## Параметры захвата

Эти параметры, включая Max/Min FPS, region и другие, аналогичны настройкам в Color Search, ML Search и Text Search. Используйте их, чтобы точнее настроить процесс поиска. [/ru/triggers/images/imagecapturetriggers](/ru/triggers/images/imagecapturetriggers)

Как и любые другие триггеры в EyeAuras, Image Search Trigger можно комбинировать с любыми другими триггерами и действиями. Благодаря этому он становится очень гибким инструментом в вашем наборе EyeAuras.

> Примечание: хотя этот триггер рассчитан на высокую производительность даже в негативных сценариях (когда изображение не найдено), стоимость поиска всё равно может быть заметной. Поэтому рекомендуется максимально уменьшать область поиска. Если в вашем сценарии изображение может поворачиваться или масштабироваться, лучше рассмотреть ML Search — поиск изображений на основе машинного обучения, устойчивый к таким преобразованиям.

## Примеры применения в играх

- **Автоматизация игрового процесса:** если в игре нужно взаимодействовать с определённой иконкой или персонажем, можно использовать Image Search Trigger для определения их появления в игровом окне. После обнаружения триггер можно связать с действием и автоматизировать часть игрового процесса.
- **Отслеживание состояний приложения:** если определённое изображение или иконка появляется только в конкретном состоянии приложения, Image Search Trigger можно использовать для отслеживания этого состояния. Например, в почтовом приложении значок может появляться только при наличии непрочитанных писем. В этом случае триггер может активировать действие и уведомить вас.
- **Определение окончания перезарядки способностей:** в играх, где способности или заклинания уходят в кулдаун после использования, можно настроить Image Search Trigger на поиск иконки "на перезарядке". Когда она исчезнет, можно запустить действие, уведомляющее, что способность снова готова.
- **Обнаружение лута:** в играх с добычей предметов можно использовать триггер для поиска иконок редкого лута. После обнаружения можно автоматически воспроизвести уведомление.
- **Распознавание противников:** если в игре нужно выбирать конкретных врагов в качестве цели, этот триггер может распознавать иконку или изображение врага, а затем использовать действие "Send Input" для наведения на цель.
- **Мониторинг полосы здоровья/маны:** используйте триггер для обнаружения иконок "мало здоровья" или "мало маны". При обнаружении можно автоматически запускать действие, например использование зелья здоровья или маны.
- **Отслеживание статусов игрока:** можно следить за такими состояниями, как "Stunned", "Silenced" и другими в MOBA- или MMORPG-играх. Когда иконка статуса исчезает, можно запускать действие для возобновления атаки или чтения заклинаний.
- **Отслеживание мини-карты:** можно находить на мини-карте определённые события, например перемещение врагов, сигналы и т. п., а затем запускать действие "Send Network Message", чтобы предупредить команду.
- **Сбор предметов и ресурсов:** в играх, где нужно собирать ресурсы или предметы, триггер может находить иконки вроде "ресурс исчерпан" или "инвентарь заполнен" и автоматически переключать вас на другую задачу.
- **Навигация по меню:** в играх со сложной навигацией по меню можно использовать Image Search Trigger для обнаружения нужных иконок или элементов интерфейса и автоматизировать переходы с помощью действия "Send Input".
- **Обнаружение AFK:** некоторые игры показывают специальную иконку или изображение, когда игрок находится в состоянии AFK. Можно настроить триггер на её обнаружение и автоматически выполнять действие, чтобы не переходить в AFK.
- **Отслеживание состояния игры:** если в игре есть разные фазы или режимы, например цикл день/ночь или безопасная зона/зона войны, можно использовать триггер для обнаружения соответствующих иконок и выполнять нужные действия. Например, в игре с циклом день/ночь можно включать "night vision", когда появляется иконка ночи.
