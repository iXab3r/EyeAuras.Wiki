# Триггер `Window Exists`

Триггер `Window Exists` предназначен для отслеживания состояния конкретного окна в системе. Он активируется, когда указанное окно найдено, и деактивируется, когда окно больше не существует или его не удаётся найти.

![Window Exists Trigger](https://s3.eyeauras.net/media/2024/05/EyeAuras_i1HqMJPIQLu7SC7C.png)

## Целевое окно

Этот параметр позволяет указать окно, за которым должен следить триггер. Для этого используется `Window Matching Expression` — выражение, описывающее нужное окно. Оно может учитывать заголовок окна, `handle`, процесс и другие характеристики, поэтому с его помощью можно достаточно гибко и точно определить, какое именно окно нужно отслеживать.

Например, если вы хотите отслеживать конкретное окно браузера, в качестве `Window Matching Expression` можно использовать заголовок веб-страницы. Пока эта страница открыта в браузере, триггер `Window Exists` будет активен. Когда страница будет закрыта или перестанет быть активной вкладкой, триггер деактивируется.

Подробнее о `Window Matching Expressions` и их правильном использовании можно прочитать в [_документации_](https://wiki.eyeauras.net/e/en/window-matching-expressions).

С помощью триггера `Window Exists` можно сделать конфигурации EyeAuras реактивными к состоянию конкретных окон и строить динамичные сценарии автоматизации с учётом текущего контекста.
