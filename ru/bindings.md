# Привязки

Привязки — это продвинутая функция EyeAuras, которая позволяет создавать динамические связи между свойствами триггеров, аур и действий. Благодаря этому система может синхронизировать свойства: если одно значение меняется, связанное значение обновляется автоматически.

Например, с помощью Bindings можно связать `Target Window` у двух разных триггеров. Если значение в одном триггере изменится, второй подстроится автоматически. Более продвинутый пример — привязать координаты последнего найденного изображения из `Image Search Trigger` к действию `Send Input`. Тогда при выполнении `Send Input` клик произойдёт по последнему найденному изображению.

### Как это выглядит в интерфейсе

У большинства свойств, которые можно настраивать через интерфейс, справа от элемента управления есть небольшой чёрный прямоугольник. Это означает, что свойство пока не привязано. Если нажать на этот прямоугольник, откроется `Binding Editor`.

В `Binding Editor` отображается дерево аур (`Aura Tree`) с доступными для привязки аурами, папками и другими объектами. Когда вы выбираете триггер или папку, интерфейс показывает список триггеров, действий и их свойств, которые можно привязать.

### Технические детали

В EyeAuras используется собственный механизм на базе выражений C# и `INotifyPropertyChanged` для создания связей между объектами. Когда обновляется привязанное свойство, система автоматически обновляет и то свойство, которое к нему привязано.

### Практическое применение и дальнейшее развитие

Bindings особенно полезны в сочетании со скриптингом EyeAuras. Вместе они позволяют создавать динамические и адаптивные оверлеи, которые реагируют на изменения в реальном времени.

Один из сильных сценариев использования — привязать координаты оверлея к области захвата (`capture region`) в `Image Search` или `Color Search`. Если вы переместите оверлей, область захвата обновится автоматически. Это даёт очень высокий уровень гибкости в настройке.

Несмотря на уже широкие возможности, функции Bindings в EyeAuras могли бы пригодиться `Conditional Bindings` и преобразования данных, чтобы сделать систему ещё гибче. Также в будущем EyeAuras планирует улучшить взаимодействие между Bindings и функцией `Custom UI`.

Например, функция `Binding to Variables` позволяет создать привязку к конкретной переменной — например, привязать `Target FPS` к переменной `userTargetFPS`. После этого можно использовать слайдер в `Custom UI`, чтобы менять значение этой переменной. Так получается динамический и интерактивный интерфейс, в котором изменения в реальном времени применяются без лишних действий.

### Bindings: вопросы и ответы

**Q: Можно ли с помощью bindings связывать объекты из разных профилей EyeAuras?**

- A: Нет. Разные профили в EyeAuras означают разные запущенные инстансы. Bindings работают только в рамках одного инстанса.

**Q: Влияют ли bindings на производительность программы?**

- A: Нет. Bindings сделаны достаточно эффективно и не оказывают заметного влияния на производительность EyeAuras.

**Q: Что происходит, если в binding возникает ошибка?**

- A: Если в binding возникает ошибка, чёрный прямоугольник рядом со свойством в интерфейсе становится красным. Причиной может быть удаление источника привязки или несовместимость типов данных.

**Q: Можно ли использовать bindings для синхронизации свойств между несколькими триггерами?**

- A: Да. Один из основных сценариев использования bindings — синхронизация свойств между разными триггерами, действиями или аурами.

**Q: Можете привести несколько примеров использования bindings?**

- A: Да.
  - Свяжите `Target FPS` сразу у нескольких триггеров. Если изменить значение в одном месте, оно обновится везде.
  - Привяжите координаты последнего найденного изображения к координатам `Send Input`. Тогда `Send Input` будет кликать по найденному изображению.
  - Привяжите координаты оверлея к области захвата в `Image Search` / `Color Search`. Когда вы перемещаете оверлей, область захвата тоже обновляется.

**Q: Передаются ли bindings при шаринге ауры?**

- A: Да. Когда аура передаётся, её bindings тоже включаются.

**Q: Что происходит с bindings при импорте ауры?**

- A: При импорте ауры bindings обрабатываются так же, как и ссылки, и сохраняются.

**Q: Можно ли использовать bindings из `Custom UI`?**

- A: Да. Bindings можно использовать внутри функции `Custom UI` в EyeAuras.

**Q: Есть ли у bindings механизмы обработки или исправления ошибок?**

- A: На данный момент bindings либо работают, либо нет. Встроенного механизма обработки или исправления ошибок сейчас нет, но это может измениться в будущих обновлениях.

**Q: Где можно увидеть bindings визуально?**

- A: Bindings отображаются в оверлее `Dependencies Viewer` в интерфейсе EyeAuras.
