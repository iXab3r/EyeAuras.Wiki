# Перенос папки `%appdata%\CVATAAT` на другой диск с помощью символической ссылки

В этом руководстве пошагово показано, как создать символическую ссылку для каталога `%appdata%\CVATAAT`, чтобы перенаправить его в другую папку на другом диске. Это может быть полезно, если в `%appdata%`, расположенной на системном диске, не хватает свободного места.

## Что понадобится

- Права администратора в Windows.
- Путь к папке назначения, куда вы хотите перенаправить содержимое `%appdata%\CVATAAT`.

## Шаг 1. Откройте каталог AppData

В `Explorer` введите в адресную строку следующий путь и нажмите Enter:

```
%appdata%
```

## Шаг 2. Подготовьте папку `CVATAAT`

- Проверьте, существует ли папка `CVATAAT` в `%appdata%`.
- Если папки **нет** — **создайте** её.
- Если папка **есть** — **переместите** её в нужное место, например в `D:\CVATAAT`.

## Шаг 3. Запустите командную строку от имени администратора

Нажмите `Win + S`, чтобы открыть поиск. Введите `cmd` или `Command Prompt`. Затем нажмите правой кнопкой мыши по `Command Prompt` и выберите `Run as administrator`.

## Шаг 4. Создайте символическую ссылку

Используйте команду `mklink`, чтобы создать символическую ссылку. Замените `D:\CVATAAT` на нужный вам путь назначения.

Выполните следующую команду:

```
mklink /D "%appdata%\CVATAAT" "D:\CVATAAT"
```

Где:

- `/D` — создаёт символическую ссылку на каталог.
- `%appdata%\CVATAAT` — место, где будет создана ссылка.
- `D:\CVATAAT` — путь к папке, на которую должна указывать ссылка.

После выполнения вы должны увидеть примерно такой вывод: `symbolic link created for CVATAAT <<===>> D:\CVATAAT`

## Готово

Теперь все данные, которые записываются в `%appdata%\CVATAAT`, будут перенаправляться в `D:\CVATAAT` или в указанную вами папку. В большинстве случаев такая схема совместима с обычным ПО, но для надёжности лучше проверить работу именно с теми приложениями, которые вы планируете использовать.

---

Помните: символические ссылки — мощный инструмент, и использовать их нужно осторожно. Неправильная настройка может привести к потере данных или нестабильной работе системы. Перед изменением системных каталогов и ссылок обязательно убедитесь, что у вас есть резервные копии.
