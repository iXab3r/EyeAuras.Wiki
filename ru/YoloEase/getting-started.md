# YoloEase

```text
старт: 5-10 кадров руками -> первая модель
                            |
                            v
десятки/сотни кадров -> авторазметка -> проверка и исправления
        ^                                      |
        |                                      v
        +---------- новая модель <- Trainer <--+
                            |
                            v
                 .pt/.onnx -> EyeAuras или другая программа
```

Задача YoloEase — помочь обучить YOLO-модель, которая ищет нужные объекты на ваших кадрах. На выходе получаются обычные веса `.pt` для дальнейшего обучения и файл `.onnx` для запуска модели без Python.

EyeAuras в этом руководстве — пример программы, где используется готовая модель. В примере с AimTrainer.io модель ищет цели `tgt` и кнопку `btn`, а EyeAuras потом загружает ONNX в `ML Search` и кликает по найденным целям через Behavior Tree.

![Start screen](https://s3.eyeauras.net/media/2026/05/YoloEase_egOi46KQfu.png)

## Минигайд на практике

Не начинайте с большого идеального набора данных. Самый быстрый путь — маленькая ручная затравка, первая модель, затем более крупные задачи с авторазметкой.

1. Возьмите первые `5-10` кадров.
2. Разметьте их руками.
3. Запустите обучение.
4. Пока первая модель обучается, добавьте еще немного ручной разметки, если успеваете.
5. Когда первая модель готова, используйте ее для авторазметки следующей задачи.
6. Если авторазметка уже помогает, увеличивайте размер задач: сначала десятки кадров, затем сотни.
7. Проверяйте предложения модели, исправляйте ошибки и снова отправляйте задачу в обучение.

Так после каждого цикла появляется новая версия модели. Сначала она слабая, затем помогает всё больше, а через несколько поколений вы уже не рисуете каждую рамку с нуля, а проверяете и исправляете авторазметку.

Подробнее про этот цикл: [разметка и обучение](/ru/YoloEase/annotate-and-train), [Trainer](/ru/YoloEase/features/trainer), [редактор разметки](/ru/YoloEase/features/annotation-editor).

## Что создается по дороге

| Шаг | Что появляется |
| --- | --- |
| Видео или папка | Исходные кадры, из которых будет собран набор данных. |
| `Extract frames` | Копии кадров внутри проекта YoloEase. |
| `Create Task` | Задача разметки: сначала маленькая, позже на десятки или сотни кадров. |
| Редактор разметки | Рамки объектов и метки вроде `tgt` или `btn`. |
| `Finish Job` | Завершенная задача, которую можно брать в обучение. |
| `Trainer` | YOLO-набор данных, обучение, предсказания и файлы модели. |
| `Open` у модели | Папка с `.pt`, `.onnx`, графиками и результатами обучения. |
| EyeAuras или другая программа | Реальная автоматизация, которая использует готовый `.onnx`. |

## Короткий словарь

| Термин | Смысл |
| --- | --- |
| Кадр | Одно изображение из видео или папки. |
| Метка или класс | Имя типа объекта: `tgt`, `btn`, `enemy`, `loot`. |
| Задача | Партия кадров, которую удобно обработать за один заход. |
| `.pt` | PyTorch-веса для обучения и дообучения через YOLO/Ultralytics. |
| `.onnx` | Файл для запуска модели в EyeAuras или другом совместимом движке. |
| Предсказание | Объекты, которые нашла модель: рамка, класс и уверенность. |
| Авторазметка | Использование уже обученной модели, чтобы быстрее разметить новые кадры. |
| Ложное срабатывание | Модель увидела объект там, где его нет. |
| Пропуск | Модель не нашла объект, который должна была найти. |

## Что получится

В конце у вас будет:

- проект YoloEase с исходными кадрами, задачами, разметкой и историей обучения;
- `.pt` веса для дальнейшего обучения в YOLO/Ultralytics;
- `.onnx` модель для запуска без Python-окружения;
- при желании — готовая логика в EyeAuras, которая использует `ML Search` и Behavior Tree.

## Основной цикл

1. Скачайте свежую сборку с [GitHub releases](https://github.com/iXab3r/YoloEase/releases/) и запустите YoloEase.
2. Создайте проект через `New...`. YoloEase покажет обычный диалог сохранения `.yeproj` и создаст рядом папку проекта. Подробнее: [подготовка данных](/ru/YoloEase/prepare-data).
3. Откройте `Prerequisites`, нажмите `Check all`, затем `Install missing`. Подробнее: [предварительные компоненты](/ru/YoloEase/prerequisites).
4. Добавьте метки, например `btn` и `tgt`. Подробнее: [подключение данных](/ru/YoloEase/features/data-sources).
5. Добавьте видео или папку с изображениями. Подробнее: [подключение данных](/ru/YoloEase/features/data-sources).
6. Если источник видео, извлеките кадры через `Extract frames`. Подробнее: [извлечение кадров из видео](/ru/YoloEase/features/video-splitter).
7. В `Trainer` создайте первую задачу. Подробнее: [Trainer](/ru/YoloEase/features/trainer).
8. Разметьте первые кадры вручную и обязательно нажмите `Finish Job`. Подробнее: [редактор разметки](/ru/YoloEase/features/annotation-editor).
9. Запустите `Start automatic training` и посмотрите метрики. Подробнее: [разметка и обучение](/ru/YoloEase/annotate-and-train).
10. Используйте предсказания и авторазметку для следующих задач. Подробнее: [редактор разметки](/ru/YoloEase/features/annotation-editor) и [Trainer](/ru/YoloEase/features/trainer).
11. При необходимости включите аугментации. Подробнее: [аугментации](/ru/YoloEase/features/augmentations).
12. Разберитесь, какие файлы получились после обучения. Подробнее: [YOLO ONNX и веса моделей](/ru/YoloEase/features/yolo-onnx).
13. Подключите финальный `.onnx` в EyeAuras или другой совместимый движок распознавания. Подробнее: [EyeAuras интеграция](/ru/YoloEase/features/eyeauras-integration).
