# Мини-приложение с собственным AI-чатом

| Поле | Значение |
| --- | --- |
| Сложность | Средняя или высокая |
| Пример | Редактор скриптов или mini-app в стиле EyePad со встроенным AI-чатом. |
| Итоговая цель | Добавить чат-панель, которая отвечает на вопросы, понимает контекст редактора и может вызывать разрешённые инструменты. |
| Основные технологии | AI runtime, AI chat UI, AI tools/plugins, Blazor |

## Когда использовать

- Вы делаете mini-app или редактор и хотите встроить в него AI-чат.
- Чат должен понимать, какой скрипт, вкладку, ошибку или выделение сейчас видит пользователь.
- Чат может вызывать инструменты, например поиск по документации, диагностику или безопасные действия в редакторе.

## Как это обычно выглядит

- Разместите чат-панель рядом с редактором или в отдельном tool window.
- Собирайте AI-сессию из того контекста редактора, который пользователь реально видит.
- Подключайте инструменты постепенно: чтение выделенного текста, чтение диагностик, поиск по документации, а затем при необходимости предложение или применение правок.
- Перед любыми изменениями файлов, скриптов или состояния приложения запрашивайте подтверждение.

## Что нужно решить заранее

- Этот чат привязан к одной вкладке редактора, одному workspace или ко всему mini-app?
- Какой контекст включается автоматически?
- Какие инструменты работают только на чтение?
- Для каких инструментов нужно явное подтверждение?
- Нужен ли ассистенту поиск по документации, или достаточно только локального контекста редактора?

## Какие API посмотреть

- `AiEngine`
- `IAiChatSession`
- `IAiChatSessionHandle`
- `AiSessionOptions`
- `AiChatViewModel`
- `AiChatComponent`
- `AiKernelPlugin`
- `IAiPluginHost`
- `AiFunctionInvocationPolicy`
- `AiDocsKnowledgeBasePlugin`
- `IBm25KnowledgeBase`

## Чего избегать

- Не делайте поиск по документации обязательным для каждого AI-чата.
- Не создавайте инструменты, которые меняют файлы или данные приложения без понятных названий и явного механизма подтверждения.
- Не позволяйте ассистенту гадать о скрытых данных редактора.
- Не рассчитывайте, что в обычных helper/plugin-классах будут доступны globals, существующие только в script runtime.
- Не сохраняйте активные session id и временные данные запросов.
