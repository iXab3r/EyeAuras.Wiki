# Бот на Memory API с интерфейсом ImGui

| Поле | Значение |
| --- | --- |
| Сложность | Высокая |
| Пример | Игровой бот в стиле `BankaBot` с живой панелью управления на ImGui. |
| Конечная цель | Собрать бота, который читает данные из другого процесса, показывает статус и элементы управления в ImGui и при необходимости позволяет пользователям добавлять свои скрипты. |
| Ключевые технологии | Memory API, ImGui SDK, встроенные скрипты, выбор окна и процесса |

## Когда использовать

- Если вы делаете именно бота, а не разовый макрос.
- Если бот читает память другого процесса.
- Если пользователю нужна постоянная живая панель с кнопками, статусом, настройками и логами.
- Если пользователям может понадобиться добавлять свои скрипты прямо внутри бота.

## Общая схема

- Используйте сервисы Memory API для чтения процесса.
- Держите чтение памяти и данные бота отдельно от кода рендера ImGui.
- Используйте ImGui как основной интерфейс бота, который всегда открыт.
- Выносите тяжёлые редакторы и длинные формы в окна Blazor только если в ImGui становится тесно.
- Добавляйте встроенные скрипты только тогда, когда пользователям действительно нужна настройка поведения во время работы бота.

## Что нужно решить заранее

- Какой backend Memory API будет поддерживать бот.
- Как пользователь будет выбирать окно игры или процесс.
- Где будет жить логика бота — в C# сервисах, Behavior Trees, aura actions или пользовательских скриптах.
- Будет ли интерфейс только на ImGui или на ImGui с дополнительными окнами-редакторами на Blazor.

## Какие API посмотреть

- `ScriptContainerExtension`
- `IWindowHandle`
- `IWindowSelector`
- `IWindowHandleProvider`
- `IProcess`
- `IImGuiExperimentalApi`
- `ICanBeDisplayedInImGui`
- `IImGuiWindowContent`
- `ImGuiWindowManager`
- `AuraScriptRunner<TSandbox>`
- `AuraScriptSandbox`

## Чего избегать

- Чтения памяти напрямую внутри render-кода.
- Собственной системы скриптов, если пользователям на самом деле не нужны runtime-расширения.
- Смешивания данных из разных подключений к процессу.
- Предположения, что ImGui встроен по умолчанию — он поставляется отдельным SDK-пакетом.
- Сохранения volatile handle-объектов и живых объектов процесса.
