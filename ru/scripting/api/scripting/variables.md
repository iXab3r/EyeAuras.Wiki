# Карта переменных в EyeAuras

Справочная карта по переменным, доступным из скриптов, типизированным обращениям к переменным, иерархии переменных аур и папок, `default.*` переменным и реактивным подпискам на изменения.

## Для чего это нужно

- Хранить небольшие настройки вне локальных переменных C#.
- Делиться значениями между скриптами, аурами, папками и поведением.
- Безопасно читать и записывать типизированные значения переменных.
- Отслеживать изменения переменных.
- Использовать значения папки как значения по умолчанию для вложенных аур.
- Понять, почему чтение переменной вернуло значение по умолчанию.
- Отличать поля скрипта на время выполнения от постоянных переменных ауры или папки.

## Модель работы

- Локальные переменные C# живут только внутри одного метода или одного запуска.
- Поля и свойства класса скрипта живут до перекомпиляции скрипта или пересоздания sandbox.
- Переменные ауры, папки и скрипта хранятся в контейнерах переменных EyeAuras.
- Переменные могут быть видны в UI у той ауры или папки, которой они принадлежат.
- `IVariablesScriptingApi` — API переменных со стороны скрипта.
- `ScriptVariable<T>` — типизированный доступ к именованной переменной.
- Ауры и папки реализуют контейнеры переменных через `IHasVariables`.
- Переменные папки могут работать как наследуемые значения по умолчанию для вложенных элементов.
- Переменные вида `default.*` обычно используют для настройки поведения подсистем по умолчанию вниз по дереву аур.

## Что есть в API

- `IVariablesScriptingApi` — основной API переменных для скриптов.
- `Variables["name"]` — прямой доступ через индексатор как к `object`.
- `Variables.Get<T>("name")` — типизированная переменная для текущего скрипта или контекста.
- `Variables.Get<T>(path, "name")` — типизированная переменная из другой ауры или папки.
- `Variables.Get<T>(IHasVariables, "name")` — типизированная переменная из известного источника.
- `ScriptVariable<T>` — типизированный доступ к переменной.
- `ScriptVariable<T>.Value` — безопасное чтение и запись; возвращает `default(T)`, если значение отсутствует, равно `null` или несовместимо по типу.
- `ScriptVariable<T>.HasValue` — `true`, если запись существует в хранилище.
- `ScriptVariable<T>.TryGetValue(out T value)` — строгое чтение без исключений.
- `ScriptVariable<T>.GetOrThrow()` — строгое чтение с исключением для обязательных значений.
- `ScriptVariable<T>.Set(value)` — удобная fluent-запись.
- `ScriptVariable<T>.Remove()` — удаляет запись.
- `ScriptVariable<T>.Listen()` — observable-поток типизированных значений.
- `IBindingValueConverterRepository` — поддержка преобразований, которая используется при типизированном чтении.

## Семантика значений

- `Value` никогда не выбрасывает исключение.
- `Value` возвращает `default(T)`, если переменной не существует.
- `Value` возвращает `default(T)`, если сохранённое значение нельзя преобразовать.
- `HasValue` означает, что запись существует, даже если сохранённое значение равно `null`.
- `TryGetValue` возвращает `false`, если значение отсутствует, равно `null` или несовместимо по типу.
- `GetOrThrow` нужен для обязательных значений, когда отсутствие или несовпадение типа считается ошибкой.
- `Remove` удаляет запись, а слушатели получают сброс к `default(T)`.
- `Listen` сначала отдаёт текущее значение, затем только отличающиеся изменения.

## Иерархия

- Переменные аур и папок образуют древовидную иерархию настроек.
- Дочерний элемент может читать унаследованные значения из родительских папок.
- Дочерний элемент может локально переопределить родительское значение.
- Переменные на уровне папки удобно использовать для целевых окон, режима ввода, общих имён профилей, задержек и других групповых настроек.
- Имена вроде `default.WindowSelector.TargetWindow` и других `default.*` могут влиять на значения подсистем по умолчанию для вложенных элементов.

## Типовые сценарии

### Необязательная настройка

1. Получите `ScriptVariable<T>`.
2. Используйте `TryGetValue`.
3. Если метод вернул `false`, подставьте локальное значение по умолчанию.

### Обязательная настройка

1. Получите `ScriptVariable<T>`.
2. Используйте `GetOrThrow`.
3. Прерывайте выполнение сразу, если значение отсутствует или некорректно.

### Общая конфигурация

1. Запишите значение в папке или конфигурационной ауре.
2. Читайте его из рабочих скриптов через path или `IHasVariables`.
3. При необходимости переопределяйте его ближе к дочерней ауре.

### Реактивный UI

1. Получите `ScriptVariable<T>`.
2. Подпишитесь через `Listen()`.
3. Добавьте подписку во время жизни текущего owner.
4. Обновляйте UI или локальное состояние при изменении значений.

## Что предпочитать

- Предпочитайте `ScriptVariable<T>`, а не прямой доступ через индексатор как к `object`.
- Для необязательных значений предпочитайте `TryGetValue`.
- Для обязательной конфигурации предпочитайте `GetOrThrow`.
- Для общих настроек связанных аур предпочитайте переменные папки.
- Используйте понятные имена и единый стиль или префиксы.
- Освобождайте подписки `Listen()` вместе со временем жизни владельца.

## Чего избегать

- Не считайте `Value == default(T)` доказательством того, что переменная отсутствует.
- Не используйте прямой доступ через индексатор там, где важна типобезопасность.
- Не храните в переменных живые handles, окна, процессы или disposable-объекты.
- Не злоупотребляйте глобальными и общими переменными для значений, которым место в локальном состоянии.
- Не считайте имена `default.*` безобидными — они могут влиять на вложенные системы.
