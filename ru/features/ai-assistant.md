# Что это такое
В EyeAuras появился встроенный **AI Assistant**.

Это отдельная вкладка внутри программы, в которой можно:

- задавать вопросы про сам EyeAuras
- искать информацию по документации
- позже получать помощь с написанием скриптов, аур и автоматизации прямо внутри программы

Главная идея очень простая: чтобы AI был не где-то "в соседней вкладке браузера", а прямо внутри EyeAuras.

![AI Chat Panel](https://s3.eyeauras.net/media/2026/03/EyeAuras_DzRWqPXjls.png)

# Что умеет уже сейчас
Сейчас самый полезный и практический сценарий - это **вопросы по документации EyeAuras**.

То есть можно спросить:

- как сделать что-то в EyeAuras
- где находится нужная настройка
- как работает та или иная фича
- какие есть варианты решения через ауры, макросы, деревья или скрипты

Если база документов включена, AI старается отвечать с опорой на wiki и показывать, откуда взята информация.

![AI Chat Example](https://s3.eyeauras.net/media/2026/03/EyeAuras_xKafNwKD2V.png)

## Что спросить в первую очередь
Если хочется просто попробовать, вот несколько хороших первых запросов:

- `Как сделать ауру, которая ищет картинку на экране?`
- `Как получить координаты найденной картинки?`
- `Как упаковать мой проект в пак?`
- `Что такое EyePad и чем он отличается от обычного режима?`
- `Как включить саблицензии для моего мини-приложения?`

Чем конкретнее вопрос, тем полезнее обычно получается ответ.

# Как начать
## 1. Откройте вкладку `AI`
В новом интерфейсе EyeAuras у вас есть отдельная вкладка `AI`.

## 2. Выберите профиль
Профиль - это просто набор настроек, который говорит EyeAuras:

- к какому AI-провайдеру подключаться
- какую модель использовать
- какой endpoint использовать
- откуда брать ключ

## 3. Убедитесь, что рядом горит `Key Available`
Если профиль настроен правильно, справа будет зеленый статус `Key Available`.

Если там `Unavailable`, значит EyeAuras пока не смог получить ключ или профиль настроен не полностью.

## 4. Нажмите `Start Session`
После этого откроется AI-чат, и можно задавать вопросы.

# Профили AI
Все настройки подключения собраны в окне `Manage Profiles`.

Там можно создать несколько профилей под разные сценарии. Например:

- один под OpenAI
- один под локальную Ollama
- один под EyeAuras Gateway

Это удобно, потому что потом между ними можно быстро переключаться прямо из AI-вкладки.

![AI Profiles - OpenAI](https://s3.eyeauras.net/media/2026/03/EyeAuras_ZwkvBDfsZA.png)

## Самые простые схемы настройки
Ниже три основные схемы, которые сейчас действительно стоит знать обычному пользователю.

### 1. Прямой API key
Это самый прямой вариант.

Вы просто вставляете ключ в поле `API Key`, и EyeAuras использует его напрямую.

Плюсы:

- все максимально прозрачно
- вы полностью контролируете свой бюджет
- ничего дополнительного на стороне EyeAuras не требуется

Минусы:

- ключ хранится именно в профиле
- обычно удобнее все-таки использовать переменную окружения

### 2. Ключ через переменную окружения
Это рекомендуемый способ, если у вас уже есть свой OpenAI key.

Вместо самого ключа вы указываете ссылку на переменную окружения, например:

```text
%OPENAI_API_KEY%
```

Тогда EyeAuras возьмет ключ оттуда.

Плюсы:

- ключ не лежит в профиле в открытом виде
- проще переиспользовать один и тот же ключ в нескольких профилях
- это более удобный и аккуратный способ настройки

### 3. EyeAuras Token через EyeAuras Gateway
Это отдельный вариант для тех, кто не хочет возиться с прямым OpenAI key или кому нужен доступ через инфраструктуру EyeAuras.

На текущий момент для этого нужно:

- в `API Key` указать `%EYEAURAS_TOKEN%`
- в `Endpoint` указать `https://eu.openai.eyeauras.net`

Пример такого профиля:

![AI Profiles - EyeAuras Gateway](https://s3.eyeauras.net/media/2026/03/EyeAuras_ZvwDEL4jNj.png)

Важно:

- сейчас этот доступ открыт **выборочно**
- чтобы подключить его, нужно связаться с **Xab3r**
- после окончания альфы этот сценарий станет доступен всем
- тогда же появятся и бесплатные лимиты
- сами лимиты еще будут подбираться по ходу дела

И еще один важный момент: это именно **опция**, а не обязательная схема.

Если вам удобнее полностью контролировать расходы самостоятельно, всегда можно использовать **свой собственный ключ** и не зависеть от EyeAuras Gateway.

Также стоит иметь в виду, что текущий адрес gateway временный. Позже endpoint поменяется, а настройка должна стать проще.

## OpenAI-compatible и локальная Ollama
Кроме OpenAI, EyeAuras умеет работать и с OpenAI-compatible API.

Самый понятный пример - локальная `Ollama`.

Типовая схема выглядит так:

- `Endpoint`: `http://localhost:11434/v1`
- `Model Id`: например `llama3`
- `API Key`: не нужен

Это удобно, если вы хотите:

- запускать модель локально на своем компьютере
- не зависеть от внешнего API
- экспериментировать без отдельных расходов на облачный AI

Если профиль настроен на `Ollama`, EyeAuras не требует API key и просто работает через указанный endpoint.

# Зачем вообще нужен EyeAuras Gateway
На первый взгляд может показаться, что проще всегда подключать только свой OpenAI key. Во многих случаях так и есть.

Но у EyeAuras Gateway есть свои задачи:

- дать базовый и более простой вход для тех, кто пользуется AI редко и не хочет сразу заводить отдельный аккаунт на платформе OpenAI
- упростить доступ к AI там, где прямое использование OpenAI может быть неудобным, например из-за блокировок
- в будущем позволить просто пополнять баланс EyeAuras вместо отдельной настройки биллинга на стороне OpenAI

Сейчас это все еще в стадии строительства, но общий смысл именно такой: сделать AI внутри EyeAuras проще и ближе к обычному пользователю.

# Что означают настройки профиля
Ниже краткое объяснение полей из менеджера профилей.

## `Name`
Просто имя профиля, под которым он будет виден в списке.

Лучше называть профили понятно, например:

- `OpenAI gpt-5.4`
- `OpenAI cheap`
- `Gateway`
- `Ollama local`

## `API Key`
Источник секрета, который будет использоваться для запросов.

Это может быть:

- сам ключ
- переменная окружения, например `%OPENAI_API_KEY%`
- `%EYEAURAS_TOKEN%` для сценария с EyeAuras Gateway

## `Endpoint`
Адрес API.

Обычно его лучше **не трогать**, если вы используете обычный OpenAI.

Но он нужен для таких сценариев:

- EyeAuras Gateway
- Ollama
- собственный proxy
- любой другой OpenAI-compatible endpoint

## `Model Id`
Идентификатор модели, например `gpt-5.4`, `gpt-5.4-nano` или `llama3`.

Если не уверены, лучше оставить рекомендованное значение из шаблона.

## `API Type`
Определяет, через какой API EyeAuras будет отправлять запросы.

Для обычного пользователя здесь лучше придерживаться шаблона по умолчанию.

Проще всего думать так:

- `Responses` - более современный вариант
- `ChatCompletions` - режим совместимости

## `Reasoning Effort`
Определяет, насколько глубоко модель будет "думать" перед ответом.

Если совсем просто:

- меньше reasoning = быстрее и дешевле
- больше reasoning = потенциально умнее, но медленнее и дороже

Если не хочется разбираться, оставляйте значение из шаблона.

## `Max Context Tokens`
Служебное поле для контроля размера контекста.

Обычному пользователю его почти всегда можно не трогать.

## `Function Calling`
Разрешает модели использовать встроенные инструменты и плагины EyeAuras.

Обычно это лучше оставить включенным, потому что именно так AI получает доступ к документации и другим возможностям.

## Статус `Key Available`
Это быстрый индикатор, что профиль готов к работе.

Если статус зеленый, EyeAuras смог понять, откуда брать ключ или как подключаться к endpoint.

Если статус красный, стоит проверить:

- правильно ли написана переменная окружения
- доступен ли EyeAuras token
- указан ли endpoint для Ollama или Gateway

# Что реально работает уже сейчас
## 1. Поиск по документации
На текущий момент это главный практический режим.

EyeAuras может скачать и проиндексировать документацию, а потом использовать ее как локальную knowledge base для ответов.

Поэтому AI особенно полезен, когда вы хотите быстро понять:

- как работает конкретная функция
- где искать нужную статью
- какой из подходов лучше для вашей задачи

## 2. Базовый in-app AI chat
AI уже работает прямо внутри EyeAuras, а не только как внешний эксперимент.

Это значит, что сама платформа под in-app AI уже есть:

- вкладка
- профили
- запуск сессии
- чат
- плагины

И дальше на этот слой будут наращиваться более прикладные сценарии.

## 3. Scripting plugin - уже скоро
Плагин, который будет помогать писать **скрипты** прямо из EyeAuras, сейчас находится в **закрытом тестировании**.

План по времени:

- `Coming Soon`
- ориентир: **early April 2026**

Именно он должен стать следующим большим шагом после текущего docs-режима.

## 4. Другие плагины уже есть, но пока не дописаны
Внутри системы уже есть направление на более глубокие плагины, включая автоматизацию самого EyeAuras.

Проще говоря, AI внутри программы постепенно движется от режима:

- `ответь мне по документации`

к режиму:

- `помоги мне сделать это прямо внутри EyeAuras`

И это касается не только самой программы. Этот же слой дальше смогут использовать и **авторы mini-app**, чтобы собирать продукты, в которых AI уже встроен по умолчанию и доступен пользователю почти без дополнительной возни.

# Частые вопросы
## Нужен ли мне аккаунт OpenAI
Не обязательно.

Есть три базовых варианта:

- использовать свой OpenAI key
- использовать локальную Ollama
- подключиться к EyeAuras Gateway через Xab3r, если вам открыт доступ к альфе

## Обязательно ли пользоваться EyeAuras Gateway
Нет.

Это дополнительная возможность, а не обязательный способ работы.

Если хочется полностью держать бюджет в своих руках, используйте свой собственный ключ.

## Почему у меня `Unavailable`
Обычно причина одна из этих:

- EyeAuras не нашел ключ по указанной переменной окружения
- для `%EYEAURAS_TOKEN%` сейчас нет доступного токена аккаунта
- для `Ollama` не указан или недоступен endpoint

## С чего начать обычному пользователю
Самый простой путь такой:

- открыть `Manage Profiles`
- создать профиль OpenAI
- указать `%OPENAI_API_KEY%`
- выбрать модель из шаблона
- нажать `Start Session`

Если своего ключа нет, но хочется попробовать сценарий через EyeAuras Gateway, можно связаться с [Xab3r](/contacts).
