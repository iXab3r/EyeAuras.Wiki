# Выражения для сопоставления текста

В EyeAuras выражения для сопоставления текста используются для проверки и сравнения строковых условий, которые задаёт пользователь. Сейчас доступно три типа проверок: **Regex**, **Text** и **Lambda**.

Они встречаются, например, в [Text Search](/ru/triggers/images/text-search), [Network Message](/ru/triggers/network-message), [Telegram Message](/ru/triggers/telegram-message) и некоторых других типах триггеров.

## Text Evaluator

`Text Evaluator` выполняет прямое сравнение с указанным текстом. По умолчанию сравнение чувствительно к регистру, но при необходимости это можно отключить.

![](/eyeauras_ruxbjypgds.png)

Примеры использования:

- **Точное совпадение с "Hello World"**  
  Текст для сравнения: `Hello World`  
  Совпадение будет только со строкой `"Hello World"` без каких-либо отличий.

- **Сравнение с числовой строкой**  
  Текст для сравнения: `12345`  
  Совпадение будет со строкой `"12345"`. Обратите внимание: это не то же самое, что число `12345`, введённое без кавычек.

- **Сравнение без учёта регистра**  
  Текст для сравнения: `HELLO`  
  Если отключить чувствительность к регистру, совпадут `"hello"`, `"Hello"`, `"HELLO"` и другие варианты.

- **Сравнение с целым предложением**  
  Текст для сравнения: `The quick brown fox jumps over the lazy dog`  
  Совпадение будет только с этим предложением целиком и без изменений.

## Regex Evaluator

`Regex Evaluator` позволяет использовать [регулярные выражения](https://regex101.com/) для более гибкой и сложной проверки текста.

![](/eyeauras_jb1lvsh9pz.png)

Примеры:

- **Найти любое число**  
  Regex для совпадения: `\d+`  
  Совпадение будет с любой строкой, представляющей число, например `"123"`, `"45678"` и т. д.

- **Найти любое слово**  
  Regex для совпадения: `\b\w+\b`  
  Совпадение будет с любой строкой, представляющей слово, например `"Hello"`, `"World"` и т. д.

- **Найти любые две буквы**  
  Regex для совпадения: `\b[a-zA-Z]{2}\b`  
  Совпадение будет со строкой, состоящей ровно из двух букв, например `"ab"`, `"cd"` и т. д.

- **Найти любые три цифры**  
  Regex для совпадения: `\b\d{3}\b`  
  Совпадение будет со строкой, состоящей ровно из трёх цифр, например `"123"`, `"456"` и т. д.

- **Найти слово, начинающееся с определённой буквы**  
  Regex для совпадения: `\bH\w*\b`  
  Совпадение будет с любым словом, которое начинается на `"H"`, например `"Hello"`, `"Hi"` и т. д.

- **Найти слово, заканчивающееся на определённую букву**  
  Regex для совпадения: `\b\w*e\b`  
  Совпадение будет с любым словом, которое заканчивается на `"e"`, например `"code"`, `"node"` и т. д.

## Lambda Evaluator

`Lambda Evaluator` — это более мощный вариант, который позволяет писать **C# Lambda**-выражения. По сути, этот тип проверки преобразует строку в логическое значение в зависимости от заданных вами условий, что даёт очень широкие возможности.

Например, можно проверить, пустая ли строка, содержит ли она определённые символы, или можно ли преобразовать её в число в нужном диапазоне.

![](/eyeauras_i9f6mfq9lz.png)

Примеры:

- **Проверить, пустая ли строка**  
  Lambda-выражение: `text => string.IsNullOrEmpty(text)`  
  Вернёт `True` для пустой строки или `Null`, и `False` в остальных случаях.

- **Проверить, содержит ли строка "Hello"**  
  Lambda-выражение: `text => text.Contains("Hello")`  
  Вернёт `True`, если строка содержит `"Hello"`, и `False` в противном случае.

- **Проверить, что длина строки меньше 5**  
  Lambda-выражение: `text => text.Length < 5`  
  Вернёт `True`, если длина строки меньше 5, и `False` в противном случае.

- **Проверить, попадает ли строка после преобразования в число в заданный диапазон**  
  Lambda-выражение: `text => double.Parse(text) >= 1 && double.Parse(text) <= 100`  
  Вернёт `True`, если строку можно преобразовать в число от 1 до 100, и `False` в противном случае.

## Test Mode

`Test Mode` помогает быстро проверить, как работает ваше выражение. Вы можете ввести строку и убедиться, что условие срабатывает именно так, как вы ожидаете. Это удобно для отладки и уменьшает вероятность ошибок.

Важно помнить, что все evaluators работают именно со строками. Они совместимы с любыми языками, поддерживающими Unicode. Ограничений на тип создаваемых выражений нет, а если выражение окажется некорректным, EyeAuras предупредит об этом.
