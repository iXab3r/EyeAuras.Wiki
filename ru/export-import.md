# Экспорт и импорт аур

В EyeAuras есть удобный способ делиться аурами с другими пользователями: экспортировать их, а затем импортировать по ссылке или через JSON. Можно собрать несколько аур в один пакет, загрузить его на сервер EyeAuras и отправить ссылку другим людям. Ниже — как это работает.

## Экспорт аур

Чтобы экспортировать ауру, сначала выберите её в AuraTree. После этого нажмите кнопку `Share` в правом верхнем углу окна EyeAuras или используйте `Ctrl + S`. Откроется окно `Export Aura`.

В окне `Export Aura` можно загрузить ауру на сервер EyeAuras. После загрузки вы получите URL-адрес, например: [this link](https://eyeauras.net/share/S20230604122817NvnkCMZ1X2qJ). Эту ссылку можно передать любому человеку, и он сможет импортировать вашу ауру.

Если вы хотите обновить уже опубликованную ауру, укажите существующий Share ID — тогда сервер перезапишет старую ауру новой версией, которую вы экспортируете.

Важно: ссылка является публичной. Не экспортируйте чувствительные данные, например пароли.

## Импорт аур

Чтобы импортировать ауры, нажмите ссылку `Import` на панели инструментов в левой части окна EyeAuras или используйте `Ctrl + V`, когда Aura Tree находится в фокусе. Откроется окно `Import Aura`.

Программа автоматически пытается распознать содержимое буфера обмена. Вы можете вставить либо JSON с пакетом аур, либо ссылку на страницу Share. В обоих случаях можно импортировать ауры, которые были ранее экспортированы.

## Как EyeAuras работает с ID аур

У каждой ауры есть глобальный уникальный идентификатор, уникальный среди всех аур в мире. Если при импорте возникает конфликт ID, EyeAuras предлагает два варианта решения:

- **Replace:** импортированная аура заменяет уже существующую.
- **Change ID:** у импортированной ауры меняется ID, а все ссылки внутри импортированных аур автоматически переписываются так, чтобы они оставались корректными.

## Вопросы и ответы

**В: Есть ли ограничение на количество аур в одном share/export пакете?**

- О: Да. Ограничение по размеру — `5MB` для незарегистрированных пользователей и `100MB` для зарегистрированных.

**В: Можно ли защитить общий пакет аур паролем, чтобы доступ был только у тех, кому я дам пароль?**

- О: Эта функция всё ещё находится в тестировании, но да — есть возможность `Private auras`, которая позволяет настраивать права доступа к вашему пакету аур.

**В: Если я обновлю общий пакет аур с тем же ID, старая версия сохранится или будет перезаписана?**

- О: При обновлении пакета с существующим ID старая версия будет перезаписана.

**В: Если при импорте возникает конфликт ID аур, повлияет ли смена ID у импортированной ауры на её работу или связи с другими аурами?**

- О: Если в импортируемом пакете меняется ID, все ссылки внутри этого пакета автоматически переписываются и обновляются. Ссылки в уже существующих аурах никогда не изменяются.

**В: Можно ли посмотреть содержимое пакета аур перед импортом?**

- О: Да, ауры можно предварительно просмотреть в окне Import.

**В: Можно ли делиться аурами без загрузки на центральный сервер EyeAuras, например просто передав файл напрямую?**

- О: Да. Если выбрать папку и нажать `Control+C` или `Copy`, в буфер обмена будет скопирован JSON с описанием выбранной папки или ауры. Этот JSON можно передать другому пользователю любым способом, и он сможет импортировать/вставить его в свою копию EyeAuras.

**В: Можно ли импортировать ауры из общего пакета по одной, или только весь пакет целиком?**

- О: По состоянию на June 2023 можно импортировать только весь пакет целиком.

**В: Как сохраняется структура папок при экспорте и импорте пакетов аур?**

- О: Папки импортируются с сохранением путей, примерно так же, как это работает в Explorer в Windows.

**В: Есть ли способ сохранить резервную копию всех аур в локальный файл на случай проблем?**

- О: Все ваши ауры уже периодически сохраняются в папку резервных копий `Backups`, которая находится здесь: `%appdata%\EyeAuras\release\backups`. Обычно сохраняются данные за последние 2 дня.

**В: Можно ли экспортировать или импортировать отдельные части ауры, например её triggers или actions, отдельно?**

- О: Да. Можно отдельно экспортировать настройки каждого action/trigger/overlay, нажав кнопку `Copy` в заголовке соответствующего элемента. После этого программа скопирует JSON с описанием Trigger/Overlay/Action, и вы сможете вставить его в другую ауру, сохранить у себя или отправить кому-то ещё.
